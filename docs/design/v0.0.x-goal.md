# v0.0.x Goal

The v0.0.x goal is to centralize target AI-native configuration definitions and materialize them into Claude project paths.

This phase focuses on centralized definitions, not exact reproducibility.

## Target AI

v0.0.x supports Claude only.

Supported materialization targets:

```text
.claude/skills/<name>/
.claude/agents/<name>.md
```

Unsupported in v0.0.x:

```text
.agents/
consumer.codex
Codex target materialization
```

## Source of Truth

The primary source of truth is:

```text
enozunu.kdl
```

Generated output is not source of truth. For v0.0.x, `.claude/` may be generated output.

The machine-generated derived record is:

```text
.enozunu/provenance.json
```

`provenance.json` records what was materialized.
It is not a lockfile.
It is not used as a resolution input in v0.0.x.

## Supported Source Reference

v0.0.x supports the normalized source reference shape:

```kdl
skill "git-kura" {
  git "https://github.com/tooppoo/reportage"
  branch "main"
  path ".claude/skills/git-kura"
}
```

Each source must use:

```text
git + branch + path
```

GitHub tree/blob URL shorthand is not supported.

## Supported Artifacts

A Skill source is a directory containing `SKILL.md`.
The whole directory is copied to the target path, including supporting files.

An agent source is a file. It is copied to `.claude/agents/<name>.md`.

v0.0.x does not validate whether a source was originally created for Claude. It validates the artifact shape needed for materialization.

## Manifest Shape

```kdl
enozunu config-version=1 {
  provider {
    skills {
      skill "git-kura" {
        git "https://github.com/tooppoo/reportage"
        branch "main"
        path ".claude/skills/git-kura"
      }
    }

    agents {
      agent "shell-script-reviewer" {
        git "https://github.com/tooppoo/installerer"
        branch "main"
        path ".claude/agents/shell-script-reviewer.md"
      }
    }
  }

  consumer {
    claude {
      use-skills "git-kura"
      use-agents "shell-script-reviewer"
    }
  }
}
```

## Generated Output Policy

Skill directory materialization uses replace semantics, not merge semantics.

When a Skill source removes a supporting file, that file should also disappear from the materialized target after regeneration.

Enozunu does not preserve, detect, or integrate manual edits inside generated output.

## Provenance

At materialization time, branch selectors are resolved to commit hashes.

The resolved revision is written to `.enozunu/provenance.json` with enough information to inspect the previous materialization result:

- source name
- source URL
- branch
- resolved revision
- source path
- target AI
- target path

Because branch selectors are mutable, running materialization again may produce a different result.

## Future Work

Exact revision selectors and reproducibility guarantees are planned for v0.1.x.
Tag selectors are introduced after revision selectors.

Later phases may also add:

- lockfiles
- frozen materialization
- Codex target materialization
- target digest based hand-edit detection
- generated output and manual edit reconciliation
- registry or marketplace discovery

Those are outside v0.0.x.
